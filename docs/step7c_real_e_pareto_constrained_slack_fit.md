# Step7C-real-E Pareto-Constrained Slack-Fit Local Lane Expansion

## Scope

Step7C-real-E is a sidecar-only one-step study. It does **not** integrate with the contest runtime solver, does **not** change finalizer semantics, does **not** start multi-iteration search, and does **not** use RL or trained ML.

This step follows Step7C-real-D's decision:

```text
expand_slack_fit_local_lane
```

The goal is to expand the known useful local patterns while selecting and reporting candidates with constrained Pareto dominance rather than hand-tuned bbox/soft penalties.

## Implemented candidate lane

Implemented in `src/puzzleplace/alternatives/pareto_slack_fit_edits.py`:

- `original_layout`: original-inclusive Pareto anchor.
- `hpwl_directed_local_nudge_expanded`: moves HPWL/pin-pressure blocks toward weighted net/pin targets.
- `slack_fit_insertion_expanded`: tries nearby axis-aligned local slack positions around HPWL-critical blocks.
- `nearby_slack_slot_insertion`: tries nearby diagonal/axis local slack positions around HPWL-critical blocks.
- `legacy_step7g_global_report`: report-only safety baseline; never sent to bounded-local repair.

Raw local probes are filtered only by official-like hard feasibility before becoming emitted candidates. There are no arbitrary bbox/soft scalar thresholds.

## Pareto objective vector

Each emitted candidate receives an objective vector:

```text
feasibility_rank
route_rank
official_like_cost_delta
hpwl_delta
bbox_area_delta
soft_constraint_delta
changed_block_fraction
```

Dominance is constrained:

1. feasible candidates dominate infeasible candidates;
2. non-global candidates dominate global/report-only candidates at equal feasibility;
3. among comparable candidates, lower objective deltas dominate;
4. original layout is always included in per-case comparison;
5. epsilon sensitivity is reported for numeric precision only (`0`, `1e-12`, `1e-9`, `1e-7`).

## Interpretation

Candidate expansion worked in count terms: official-like improvements rose from Step7C-real-B's 3 to 16. Route safety also held: local candidates stayed local, global probes stayed report-only, and invalid local attempts remained zero.

However, selection quality is still not strong enough for local-lane iteration. The official-like improvement density is `0.0462`, below Step7C-real-B's `0.0536`, and HPWL-to-official conversion is `0.1798`, below the B/C forensic baseline of `4 / 16 = 0.25`. Constrained Pareto correctly rejects many HPWL-only false positives, but the expanded generator produces too many dominated local probes.

## Decision

```text
refine_pareto_slack_fit_generator
```

Reason: expanded slack-fit local generation produces more official-like winners and preserves safety, but density and HPWL conversion are weaker than the promotion gate. The next implementation should refine candidate coverage and Pareto preselection before local-lane iteration.

## Next implementation direction

Recommended next step:

1. Keep the local slack-fit / HPWL lane as the main path.
2. Reuse constrained Pareto objective vectors and original-inclusive reporting.
3. Improve local probe generation so fewer emitted candidates are dominated by original.
4. Preserve all loser reporting, especially HPWL-only candidates rejected by Pareto.
5. Do not pivot to coarse region or macro closure yet; real-E shows local official-like wins exist, just too sparsely.
