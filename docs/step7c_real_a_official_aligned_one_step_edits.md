# Step7C-real-A: Official-Aligned One-Step Real Placement Edit Loop

## Plan

Step7C-real-A follows Step7C-thin's decision:

```text
promote_to_step7c_iterative_loop
```

Step7C-thin proved route-aware descriptors can become deterministic rectangle
edits, but those rectangles were still synthetic. Step7C-real-A validates the
same one-step loop on real FloorSet validation cases and target placements:

```text
Step7H descriptor
-> real block-id placement edit
-> Step7G locality route
-> route-appropriate repair/legalization proxy
-> official-aligned hard + metric report
-> original-inclusive Pareto report
```

This is still a sidecar-only one-step study, not a contest runtime change and
not a multi-iteration search.

## Scope

- sidecar only
- no contest runtime integration
- no finalizer semantics change
- no RL / trained ML
- no multi-iteration optimization
- no silent discard of regional, macro, or global candidates
- global candidates remain report-only and are not sent to bounded-local repair
- original layouts remain explicit Pareto candidates

## Implemented lanes

The minimal real-edit subset mirrors Step7C-thin:

- `original_layout`: unchanged real target-placement baseline.
- `vacancy_aware_local_insertion`: one movable real block receives a small
  hard-feasible displacement chosen from deterministic directions.
- `adjacent_region_reassignment`: a real adjacent/quadrant neighborhood is
  shifted together when a hard-feasible displacement exists; otherwise the
  candidate is preserved with `no_feasible_regional_real_edit` attribution.
- `mib_group_closure_macro`: a real MIB/cluster group is shifted together when
  feasible; otherwise it is preserved with `no_feasible_macro_real_edit`.
- `legacy_step7g_global_move`: broad disruptive report-only baseline. It never
  moves fixed/preplaced anchors and is never routed to bounded-local repair.

## Official-aligned checks

`src/puzzleplace/alternatives/real_placement_edits.py` uses existing project
structures and helpers:

- real cases from `load_validation_cases`
- real block ids and target-position boxes
- `summarize_hard_legality` for overlap, area tolerance, and fixed/preplaced
  exact handling
- baseline-bbox frame validity as a sidecar boundary/frame proxy
- `evaluate_positions` for official-like cost, HPWL gap, area gap, and soft
  violation reporting when available
- existing MIB/group/boundary soft counters for `mib_group_boundary_soft_delta`

## Run result

Generated on the Step7G/H/C-thin case set:

- Real FloorSet cases: 8 (`19`, `24`, `25`, `51`, `76`, `79`, `91`, `99`).
- Descriptor candidates consumed: 72.
- Real one-step edit candidates: 40.
- Real routes: local 21, regional 3, macro 8, global 8.
- `real_non_global_candidate_rate`: 0.8.
- `descriptor_to_real_route_stability`: 0.875.
- `invalid_local_attempt_rate`: 0.0.
- `official_like_hard_feasible_rate`: 0.8; the hard-invalid candidates are the
  intentional global report-only probes.
- Fixed/preplaced and area-tolerance violations: 0 after route handling.
- Original-inclusive Pareto front non-empty: 8 / 8 cases.
- Actual safe improvements: 5, all from local real one-block edits.
- Regional/macro candidates are preserved, but many are no-op/non-improving or
  require stronger route-specific legalizers.

## Decision options

- `promote_to_step7c_multi_iteration_sidecar`
- `refine_real_edit_generators`
- `build_route_specific_legalizers`
- `pivot_to_coarse_region_planner`
- `pivot_to_macro_level_move_generator`
- `revisit_proxy_to_real_assumptions`
- `inconclusive_due_to_official_metric_gap`

## Decision

```text
refine_real_edit_generators
```

Reason: route-aware real placement edits preserve non-global diversity, keep
invalid bounded-local attempts at zero, retain official-like hard feasibility for
all non-global lanes, and keep original-inclusive Pareto fronts non-empty.
However, metric improvements are sparse and mostly limited to one-block local
nudges; regional and macro real edits often fail construction or are
non-improving. The next Step7C sidecar work should improve real edit targeting
and/or route-specific legalizers before starting multi-iteration search.
