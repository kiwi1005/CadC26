# Step7L: Learning-Guided Topology-Demand Targets

## Scope

Step7L is a sidecar-only learning lane for target/action priors. It does not
modify contest runtime code, does not change finalizer semantics, and does not
let a model bypass deterministic legality or fresh full-case replay.

The current project bottleneck is target direction / topology-demand / objective
alignment. Step7ML-P-style full-case obstacle-aware repacking has made legality
increasingly tractable, but many legal non-case candidates still regress HPWL or
official-like cost. Step7L therefore uses ML/RL/GNN only as proposal priors; the
legalizer, replay, and constrained Pareto gate remain deterministic hard checks.

## Phase plan

1. Phase 0: audit and normalize data boundaries.
2. Phase 1: deterministic topology / wire-demand heatmap baseline.
3. Phase 2: heatmap-to-repacker candidate requests; replay is deferred until the repacker interface exists.
4. Phase 3: pure PyTorch GNN / masked action policy only if Phase 1/2 show signal.
5. Phase 4: offline RL / decision transformer only after replay data is large and diverse.

## Phase 0 artifact contract

Generated by `scripts/step7l_audit_learning_inputs.py`:

- `artifacts/research/step7l_phase0_audit.json`
- `artifacts/research/step7l_phase0_audit.md`

Phase 0 separates three record families:

- `training_layout_prior`: FloorSet training `fp_sol` labels for supervised
  target/heatmap priors.
- `validation_inference`: visible validation inputs only; polygons/metrics are
  explicitly discarded and must not appear as target labels.
- `sidecar_candidate_quality`: Step7 replay labels for candidate ranking / gate
  diagnostics only, not layout-prior training.

## Pass condition

Phase 0 can promote to the deterministic heatmap baseline only if:

- training samples load and validate the `fp_sol[..., 0:4] = [w, h, x, y]` contract;
- validation inference records with target labels are exactly zero;
- Step7 candidate-quality rows are present and marked as separate from layout-prior labels.

## Phase 2 artifact contract

Generated by `scripts/step7l_generate_heatmap_candidates.py`:

- `artifacts/research/step7l_phase2_candidate_requests.jsonl`
- `artifacts/research/step7l_phase2_request_summary.json`
- `artifacts/research/step7l_phase2_request_summary.md`

Phase 2 is request-only. It compares four request families per selected
validation case:

- `original_anchor`: one baseline anchor row for later replay comparison.
- `topology`: target windows from the blended terminal/boundary/center heatmap.
- `terminal`: target windows from pin-to-block terminal demand only.
- `union_diversified`: de-duplicated interleave of topology and terminal requests.

The generator loads visible validation inputs by loader index, discards official
validation label tensors, and records `validation_labels_discarded: true` in
request provenance. Output rows are `route_class = unrouted_request_sidecar`
except the original anchor. They are not replay claims and cannot promote a
source family until a later repacker interface performs full-case fresh replay
and Step7N-I/J constrained Pareto gating.

## Phase 2 replay / completion artifact contract

Generated by `scripts/step7l_replay_heatmap_candidates.py`:

- `artifacts/research/step7l_phase2_replay_rows.jsonl`
- `artifacts/research/step7l_phase2_summary.json`
- `artifacts/research/step7l_phase2_decision.md`
- `artifacts/research/step7l_phase2_failures_by_case.json`
- `artifacts/research/step7l_completion_report.json`
- `artifacts/research/step7l_completion_report.md`

The replay bridge is sidecar-only. It consumes Step7L request windows, uses
validation labels only at replay/evaluation time to reconstruct the baseline
geometry required by the official-like evaluator, and then tries a bounded
single-block full-case non-overlap target realization. It does not mutate contest
runtime code or finalizer semantics.

Current completion decision: `complete_step7l_deterministic_prior_and_defer_gnn_rl`.
The bridge realized 256 hard-feasible non-noop candidates with zero full-case
overlap after splice, but produced 0 fresh official-like improving rows and 0
quality-gate passes. Phase 3 GNN and Phase 4 offline-RL gates therefore remain
closed. The Step7L result is a negative but useful diagnosis: the request/replay
plumbing is complete, and the next bottleneck is objective-aligned target
direction rather than legality.
