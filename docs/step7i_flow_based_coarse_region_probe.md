# Step7I Flow-Based Coarse Region Probe

## Scope

Step7I is a sidecar-only coarse-region assignment probe. It does **not** integrate with the contest runtime solver, does **not** change finalizer semantics, does **not** replace Step7G routing, does **not** start a full global optimizer, and does **not** use RL or trained ML.

This step follows Step7C-local-iter-0's conclusion: local one-block slack-fit edits are safe but quickly starve. Step7I asks whether coarse region/topology assignment can create useful non-local candidates for those starved cases.

## Method

Implemented in `src/puzzleplace/alternatives/coarse_region_flow.py` and run by `scripts/step7i_run_flow_based_coarse_region_probe.py`.

The probe uses a deterministic approximation to flow/min-cost assignment:

1. Build a 4x4 coarse region grid from the current official-aligned target placement.
2. Compute per-region occupancy, unused capacity, fixed/preplaced area, boundary area, utilization, and overflow.
3. Build small movable block/community groups from:
   - high HPWL/terminal-pressure blocks;
   - net-community clusters;
   - bbox-owner/extreme blocks.
4. Score each group-to-region assignment with separately reported components:
   - `hpwl_community_demand`
   - `region_slack_capacity`
   - `bbox_pressure`
   - `fixed_preplaced_incompatibility`
   - `mib_group_closure_risk`
   - `boundary_ownership_compatibility`
5. Select a small deterministic set of low-cost assignments per case.
6. Apply bounded group translations toward target regions as real placement edits.
7. Route every edit with Step7G locality classification.
8. Evaluate official-like hard feasibility, HPWL, bbox, soft deltas, and Pareto status.

Global candidates remain report-only and are not sent to bounded-local repair. Original layouts remain in each Pareto comparison.

## Interpretation

The probe succeeds at producing non-global regional/macro assignment candidates rather than only local one-block moves. It also preserves Step7G route safety and original-inclusive Pareto reporting.

However, the concrete bounded group-translation edits are not yet legal: every non-original regional/macro assignment fails official-like hard feasibility after the edit. No local-starved case is recovered by an official-like improving candidate. This means the current blocker is not absence of region-assignment signal; it is the lack of a route-specific regional legalizer/repacker that can realize those assignments without overlaps or hard violations.

## Decision

```text
build_regional_legalizer
```

Reason: coarse assignment creates non-noop non-global regional/macro candidates, but they fail as raw translations. A regional legalizer is needed before judging whether the flow-style costs can improve official-like metrics.

## Next implementation direction

Recommended next step:

1. Keep the Step7I region maps and assignment cost decomposition.
2. Build a small route-specific regional legalizer/repacker for selected region-assignment candidates.
3. Re-evaluate whether the same assignments become hard-feasible and metric-relevant.
4. Do not promote a region flow planner yet; there are zero official-like improvements without legalization.
5. Do not pivot to training/retrieval or force maps until a minimal regional legalization pass has been tested.
