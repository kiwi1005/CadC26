# Step7I-R Route-Specific Regional Legalizer / Repacker Probe

## Scope

Step7I-R is a sidecar-only legalizer probe over Step7I coarse-region assignment candidates. It does **not** integrate with the contest runtime solver, does **not** change finalizer semantics, does **not** start a global optimizer, does **not** replace Step7G routing, and does **not** use RL or trained ML.

This step follows Step7I's decision:

```text
build_regional_legalizer
```

The goal is to test whether Step7I's non-noop regional/macro assignment structure can be converted into hard-feasible or legalizer-valid alternatives before judging metric upside.

## Implementation

Implemented in `src/puzzleplace/alternatives/regional_legalizer.py` and run by `scripts/step7i_r_run_regional_legalizer_probe.py`.

The probe loads `artifacts/research/step7i_route_report.json` rows and applies deterministic route-specific handling:

- `original_anchor`: preserves original-inclusive Pareto anchors.
- `global_report_only`: preserves global candidates as report-only; they are not routed to bounded-local repair.
- `slot_assignment_matching`: for pure regional rows, builds a source/target regional window, generates bounded feasible slots, and tries to assign moved blocks without overlap against fixed/static blocks.
- `supernode_closure_probe`: for macro/MIB/group-closure rows, reports that the candidate requires closure-supernode handling before expansion. These rows are not silently dropped or mislabeled as feasible regional placements.

If slot matching cannot find a legal non-overlapping move, the candidate is reported as `no_feasible_slot` and falls back to a no-op legal state for safe metric accounting. This makes hard feasibility explicit but does **not** claim a useful alternative.

## Interpretation

The bounded regional legalizer does not yet convert Step7I region assignments into useful hard-feasible alternatives. It safely prevents raw infeasible translations from contaminating metric claims, but almost all non-original candidates are either macro-closure-limited, global report-only, or no-op after slot matching.

The dominant finding is that MIB/group/macro closure is not a small side condition. Thirty of the non-local candidates require supernode-style closure handling before any regional repack can be meaningfully evaluated.

## Decision

```text
pivot_to_macro_closure_generator
```

Reason: Step7I-R does not recover local-starved cases or produce official-like improvements, and macro/group closure dominates the legalizer failure mode. The next useful probe should build macro-closure/supernode move generation or legalization rather than adding more raw regional slot matching.

## Next implementation direction

Recommended next step:

1. Build a sidecar macro-closure/supernode generator for MIB/group-connected candidate sets.
2. Evaluate closure fit and route behavior before expanding back into individual blocks.
3. Preserve Step7G route safety and original-inclusive Pareto reporting.
4. Keep regional flow assignment artifacts as diagnostic inputs, but do not promote them until macro closure can be realized legally.
