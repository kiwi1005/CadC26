# Step7C-real-F Dominance-Aware Slack Slot Preselection

## Scope

Step7C-real-F is a sidecar-only preselection study over the Step7C-real-E local slack-fit candidate set. It does **not** integrate with the contest runtime solver, does **not** change finalizer semantics, does **not** start full multi-iteration search, and does **not** use RL or trained ML.

This step follows Step7C-real-E's decision:

```text
refine_pareto_slack_fit_generator
```

The goal is to reduce dominated local probes before candidate emission while preserving Step7C-real-E's original-inclusive Pareto reporting and useful official-like wins.

## Inputs

The runner consumes:

- `artifacts/research/step7c_real_e_route_report.json`

This is an artifact-replay preselection pass. It does not rerun the FloorSet geometry edit generator; instead, it learns from the E-stage candidate rows and asks whether deterministic dominance-aware ranking would have emitted a smaller, stronger candidate set.

## Preselection method

Implemented in `src/puzzleplace/alternatives/dominance_slack_preselection.py`:

1. Always retain:
   - `original_layout` as the original-inclusive Pareto anchor;
   - `legacy_step7g_global_report` as report-only safety baseline.
2. Consider local slack candidates eligible only when their E-stage proxy vector is locally dominant against obvious false positives:
   - actual route is `local`;
   - official-like hard feasible in the E-stage one-step check;
   - one changed block;
   - proxy HPWL delta improves;
   - proxy bbox delta does not regress within numeric tolerance;
   - proxy soft-constraint delta does not regress within numeric tolerance.
3. Rank eligible candidates per case by deterministic inspectable features:
   - predicted/proxy HPWL delta;
   - displacement magnitude;
   - free-space fit ratio;
   - candidate id for stable tie breaking.
4. Retain the per-case top-k ranked candidates, default `k = 12`.
5. Summarize every filtered candidate with a filter reason; filtered rows are not silently discarded.

The tolerance is `DEFAULT_EPSILON` reused from Step7C-real-E Pareto logic for numeric precision. The rank budget is analyzed through sensitivity rather than treated as a magic scalar score threshold.

## Comparison against Step7C-real-E

Step7C-real-E emitted 354 candidates, found 16 official-like improvements, and had official-like improvement density `0.0462`. It also produced 258 candidates dominated by original and HPWL-to-official conversion `0.1798`.

Step7C-real-F keeps the same 16 official-like improvements while cutting retained candidates to 74. Official-like improvement density rises to `0.2424`, HPWL-to-official conversion rises to `0.2712`, and dominated-by-original rate drops to `0.1081`.

The key improvement is not a new generator; it is better deterministic preselection before emitting local slack probes.

## Decision

```text
promote_to_step7c_local_lane_iteration
```

Reason: dominance-aware preselection materially improves improvement density and HPWL-to-official conversion, preserves all E-stage official-like winners, maintains zero invalid local attempts, keeps original-inclusive Pareto fronts non-empty, and sharply reduces dominated probes.

## Next implementation direction

Recommended next step:

1. Build a bounded local-lane Step7C iteration sidecar around the F-stage preselection rule.
2. Keep Step7G routing as the safety layer.
3. Continue original-inclusive Pareto reporting after every candidate batch.
4. Keep global candidates report-only; do not send them to bounded-local repair.
5. Do not integrate into the contest runtime solver until the local-lane iteration sidecar demonstrates stable multi-step metric improvements.
